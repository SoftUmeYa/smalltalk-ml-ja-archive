皆さん、ごきげんよう。鷲見です。

Squeak 3.8 や Nihongo7 で、$×、$÷ など charCode が 16rA0 〜 16rFF 範囲内の
文字を含む文字列をカット＆ペーストすると文字化けが起きたり、例外があがります
（Mac OS 9、Win XP で確認）。

EncodedCharSet class >> #charFromUnicode: 冒頭の unicode < 256 を unicode <
16rA0 などと変えると、とりあえず直るようです。ご報告まで。
